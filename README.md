# tgVpnBot — Telegram-бот для администрирования VPS

Личный инструмент для управления собственным VPN-сервером прямо из Telegram. 
Вместо того чтобы заходить в браузер и открывать веб-интерфейс **3x-UI** — все административные действия 
доступны через бота: создать подключение, выдать ключ, посмотреть статистику.

## Идея

Когда у тебя свой VPS с поднятым Xray и панелью 3x-UI — рутинные задачи вроде
создания нового клиента или просмотра трафика требуют открывать браузер, вводить адрес панели, логиниться.
Этот бот автоматизирует всё через Telegram, используя REST API который 3x-UI 
предоставляет для всех операций из своего графического интерфейса.

## Как это работает

Бот общается с 3x-UI через **Spring Cloud OpenFeign** — декларативный HTTP клиент. 
При первом обращении авторизуется в панели, получает сессионный токен и сохраняет его в **PostgreSQL**. 
Повторная авторизация происходит автоматически когда токен протухает (раз в час).

Входящие сообщения от пользователей обрабатываются через **TelegramBotService**, 
который маршрутизирует команды к соответствующим сервисам.
Сессии и история сообщений хранятся в базе данных. Схема управляется через **Flyway**.

## Tech Stack

| | |
|---|---|
| Java 17 + Spring Boot 3.3 | Основа |
| Spring Cloud OpenFeign | HTTP клиент для 3x-UI REST API |
| PostgreSQL + Hibernate | Хранение токенов, сессий, пользователей |
| Flyway | Версионирование схемы БД |
| Telegram Bots API | Интеграция с Telegram |
| Maven | Сборка |
| Docker Compose | Запуск инфраструктуры |

## Статус проекта

Проект в активной разработке.

**Готово:**
- Авторизация в 3x-UI и хранение сессионного токена с автообновлением
- Базовая структура бота — приём сообщений, сессии пользователей, история
- Модели данных: User, Client, Inbound, Session, Message, TelegramLog

**В планах:**
- Создание нового inbound (подключения)
- Генерация и выдача ключей клиентам
- Просмотр статистики трафика
- Управление существующими подключениями (включить / выключить / удалить)

## Getting Started

Требования: **Docker Desktop**, собственный VPS с развёрнутым **3x-UI**.

### 1. Сертификат 3x-UI
Если сертификат не самоподписанный, то можно пропустить данный пункт.

Если твой 3x-UI использует самоподписанный SSL-сертификат, 
его нужно добавить в проект. Скачай публичный сертификат с сервера
и сохрани как `src/main/resources/cert.crt`.

> Нужен только публичный сертификат (`cert.crt`). Приватный ключ никуда передавать не нужно.

 
### 2. Переменные окружения

Создай `.env` файл в корне проекта:

```env
TG_BOT_USERNAME=your_bot_username
TG_BOT_TOKEN=your_bot_token
TG_BOT_PASSWORD=your_bot_password
XUI_BASE_URL=https://your-vps:port/path
XUI_USERNAME=admin
XUI_PASSWORD=password
```

**Вариант 1 — всё в Docker:**
```bash
git clone https://github.com/epsiloNperseI/tgVpnBot.git
cd tgVpnBot
docker-compose up --build
```

**Вариант 2 — локальная разработка:**
```bash
# Только PostgreSQL
docker-compose up -d postgres

# Запуск Spring приложения из IDE(переменные окружения)
```
